Building a Content-Based Recommender System Using Movie Data

Recommender systems are an essential part of many online platforms like Netflix, Amazon, and YouTube. They help users discover content that matches their preferences by filtering information based on patterns and user data. In this tutorial, we will build a Content-Based Recommender System using movies as an example.

What is a Content-Based Recommender System?

A content-based recommender system suggests items to a user by finding similarities between items and recommending those that are most similar to what the user has already liked or interacted with. Unlike collaborative filtering, which relies on user-user or item-item interactions, content-based filtering focuses on the attributes of the items themselves.

Project Overview

In this tutorial, we'll use a dataset of movies with metadata such as genres, descriptions, and tags. Our goal is to build a recommendation system that suggests movies based on a selected movie's content features, such as genres, cast, or movie descriptions.

By the end of this tutorial, you'll understand how to:

  1. Load and process movie data.
  2. Extract key features for content-based filtering.
  3. Implement a similarity metric (e.g., sigmoid similarity).
  4. Build a movie recommender system that provides relevant recommendations based on movie metadata.

Step 2: Merging and Cleaning the Datasets

Now that we have loaded the movie and credits datasets, we need to merge them based on the common id column. After merging, we'll remove unnecessary columns to keep our data clean and focused on the features that are most relevant for building the recommendation system.

credits_column_renamed = credits.rename(index=str, columns={"movie_id": "id"})
  movies_merge = movies.merge(credits_column_renamed, on='id')
  movies_cleaned = movies_merge.drop(columns=['homepage', 'title_x', 'title_y', 'status','production_countries'])

Explanation:

  1. Renaming and Merging:
    • We start by renaming the movie_id column in the credits dataset to id, ensuring that both datasets have a common key to merge on.
    • Next, we merge the movies and credits datasets on the id column using the merge() function. This combines the two datasets into a single DataFrame, which now contains movie metadata (from movies) and movie credits (from credits).
  2. Cleaning the Dataset:
    • After merging, we remove irrelevant columns using the drop() function. Specifically, we drop the columns:
      • homepage: The URL of the movie's homepage, which isn’t needed for our recommender system.
      • title_x and title_y: These are duplicate columns for the movie title, created during the merge process.
      • status: Whether the movie is released or not; not relevant for recommendations.
      • production_countries: Information about the countries where the movie was produced, which isn't useful in our content-based filtering.

At this point, our dataset contains only the essential columns we need for building a content-based movie recommendation system.

Step 3: Feature Extraction Using TF-IDF

In this step, we will extract key textual features from the movie data. One of the most important features in a content-based recommendation system is the plot summary (or overview) of each movie. We'll use TF-IDF (Term Frequency-Inverse Document Frequency) to convert the text data (movie overviews) into numerical values that reflect the importance of words.

The TfidfVectorizer from the scikit-learn library helps us convert the text data into a matrix of TF-IDF features. This transformation will allow us to compute the similarity between different movies based on their overviews. 

from sklearn.feature_extraction.text import TfidfVectorizer
  tfv = TfidfVectorizer(min_df=3,  max_features=None,
              strip_accents='unicode', analyzer='word',token_pattern=r'\w{1,}',
              ngram_range=(1, 3),
              stop_words = 'english')
  tfv_matrix = tfv.fit_transform(movies_cleaned_df['overview'].values.astype('U'))

Explanation:

  1. TfidfVectorizer():
    • This function converts a collection of raw text into a matrix of TF-IDF features. We customize it with several parameters:
      • min_df=3: Ignores terms that appear in fewer than 3 documents (movies) to remove very rare words.
      • max_features=None: This means we don’t set a limit on the number of features. All words that meet the min_df threshold will be used.
      • strip_accents='unicode': This removes accent marks from characters, ensuring text normalization.
      • analyzer='word': This specifies that the vectorizer will analyze the text at the word level.
      • token_pattern=r'\w{1,}': Defines the tokenization pattern, specifying that words of at least 1 character should be considered.
      • ngram_range=(1, 3): This allows the vectorizer to consider single words (unigrams), two-word combinations (bigrams), and three-word combinations (trigrams).
      • stop_words='english': This removes common English stop words (like "the", "and", "is") that do not add meaningful information.
  2. tfv.fit_transform(movies_cleaned['overview'].values.astype('U')):
    • fit_transform(): This function first fits the TF-IDF vectorizer to the data (i.e., it learns the vocabulary and calculates TF-IDF scores) and then transforms the movie overviews into a TF-IDF matrix.
    • movies_cleaned['overview'].values.astype('U'): Here, we extract the overview column from the cleaned dataset and convert it to Unicode (using .astype('U')) to handle any potential issues with non-UTF text encoding.
  3. tfv_matrix: This is the resulting matrix where each row represents a movie, and each column represents a TF-IDF score for a word (or n-gram) in the movie's overview. This matrix will later be used to calculate similarities between movies.

Step 4: Calculating Movie Similarities Using the Sigmoid Kernel

In this step, we will calculate the similarity between movies based on their TF-IDF features. We will use the Sigmoid Kernel from the scikit-learn library to compute a similarity score between all pairs of movies. The sigmoid kernel is commonly used in machine learning to map data points into a non-linear feature space, making it suitable for content-based recommendations.

from sklearn.metrics.pairwise import sigmoid_kernel
  
  # Compute the sigmoid kernel
  sig = sigmoid_kernel(tfv_matrix, tfv_matrix)
  print(sig[0])
[0.76163447 0.76159416 0.76159416 ... 0.76159416 0.76159416 0.76159416]

Explanation:

  1. sigmoid_kernel():
    • The sigmoid_kernel() function computes a similarity matrix between the input data points. In our case, it compares the TF-IDF features of each movie with every other movie, giving a similarity score between 0 and 1.
    • Input: The function takes the TF-IDF matrix (tfv_matrix) as input. The matrix is compared with itself, meaning we are calculating the similarity between all movies based on their overview.
  2. sig = sigmoid_kernel(tfv_matrix, tfv_matrix):
    • Here, the sigmoid_kernel() computes the similarity score between every pair of movies. The result is a 2D matrix (sig), where each element sig[i][j] represents the similarity score between movie i and movie j. A higher score indicates higher similarity between the movies.
  3. print(sig[0]):
    • This line prints the similarity scores for the first movie (index 0). The result is a list of similarity scores between the first movie and every other movie in the dataset, including itself (the similarity with itself will be the highest).

Step 5: Mapping Movie Titles to Indices and Finding Similar Movies

Now that we have computed the similarity scores using the sigmoid kernel, we need a way to map movie titles to their corresponding indices in the dataset. This mapping will allow us to easily access the similarity scores for any given movie and recommend similar movies.

In this step, we'll create a reverse mapping of movie indices and titles and use it to find the movies most similar to a given movie.

# Reverse mapping of indices and movie titles
  indices = pd.Series(movies_cleaned.index, index=movies_cleaned['original_title']).drop_duplicates()
  sorted(list(enumerate(sig[indices['Newlyweds']])), key=lambda x: x[1], reverse=True)

Explanation:

  1. Reverse Mapping:
    • pd.Series(movies_cleaned.index, index=movies_cleaned['original_title']).drop_duplicates():
      • This line creates a reverse mapping between the movie titles (original_title) and their corresponding indices in the movies_cleaned DataFrame. We use drop_duplicates() to avoid any duplicate entries in the mapping.
      • This mapping will be useful to find a movie’s index based on its title, which we can then use to look up its similarity scores in the sigmoid kernel matrix.
  2. indices['Newlyweds']:
    • This retrieves the index of the movie titled "Newlyweds" from the reverse mapping. In our case, it will return the index 4799.
  3. Similarity Scores:
    • sig[4799]: This retrieves the similarity scores for the movie at index 4799 (which corresponds to "Newlyweds") from the similarity matrix sig. It returns a list of similarity scores between "Newlyweds" and every other movie in the dataset.
    • list(enumerate(sig[indices['Newlyweds']])): This enumerates the similarity scores for "Newlyweds," pairing each score with the index of the corresponding movie. The result is a list of tuples where the first element is the movie's index and the second is its similarity score with "Newlyweds."
  4. Sorting the Similar Movies:
    • sorted(list(enumerate(sig[indices['Newlyweds']])), key=lambda x: x[1], reverse=True):
      • This line sorts the enumerated list of similarity scores in descending order (from most similar to least similar), using a lambda function that sorts based on the similarity score (x[1]).
      • The sorted list will help us identify the top movies that are most similar to "Newlyweds."

In the next step, we'll use this sorted list to recommend the top N similar movies to any given movie title.

Step 6: Building the Recommendation Function

Now that we have computed the similarity scores for all movies and mapped movie titles to their corresponding indices, we can build the recommendation function. This function will take a movie title as input and return the top 10 most similar movies based on the sigmoid kernel similarity scores.

def give_recomendations(title, sig=sig):
      # Get the index corresponding to original_title
      idx = indices[title]
  
      # Get the pairwsie similarity scores
      sig_scores = list(enumerate(sig[idx]))
  
      # Sort the movies
      sig_scores = sorted(sig_scores, key=lambda x: x[1], reverse=True)
  
      # Scores of the 10 most similar movies
      sig_scores = sig_scores[1:11]
  
      # Movie indices
      movie_indices = [i[0] for i in sig_scores]
  
      # Top 10 most similar movies
      return movies_cleaned['original_title'].iloc[movie_indices]

    • This extracts the indices of the top 10 most similar movies from the sorted list.
  7. movies_cleaned['original_title'].iloc[movie_indices]:
    • Finally, we use the extracted indices to return the titles of the 10 most similar movies from the movies_cleaned DataFrame.

Conclusion

In this tutorial, we successfully built a content-based recommendation system using movie overviews as the primary feature for measuring similarity. Here's a brief recap of what we covered:

  1. Data Preparation: We started by loading and merging two datasets, tmdb_5000_movies.csv and tmdb_5000_credits.csv, and cleaned the data to focus on relevant features.

  2. Feature Extraction: We utilized the TF-IDF Vectorizer to convert movie overviews into numerical representations, capturing the importance of each word in the context of the dataset.

  3. Similarity Calculation: By employing the Sigmoid Kernel, we computed a similarity matrix that quantifies how similar each movie is to every other movie based on their overviews.

  4. Mapping Titles to Indices: We created a reverse mapping to easily access movie indices using their titles, which is essential for retrieving similarity scores.

  5. Recommendation Function: Finally, we implemented a recommendation function that identifies the top 10 movies most similar to a given title, allowing users to receive tailored suggestions based on their interests.

This content-based recommendation system can be further enhanced by incorporating additional features such as genres, directors, or actors, and exploring other recommendation algorithms like collaborative filtering.

By understanding the fundamentals of how recommendation systems work, you can apply similar principles to various domains, including e-commerce, music, and books, to provide personalized experiences for users.
